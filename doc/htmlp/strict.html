Transitional and Strict modes

Since version 4.09, Camlp5 can be installed in two modes: transitional mode and strict mode.

This notion has been introduced to ensure backward compatibility and allow the use of a new quotation kit "q_ast.cmo", defining Camlp5 syntax tree quotations in user syntax (with all its possible extensions).

Here are the differences with the two modes:

Transitional
Compatibility
The syntax tree is fully compatible with the previous versions of Camlp5, no changes has to be done in users' programs.
Quotation kit "q_ast.cmo"
The antiquotations are not available: when used, a syntax error message is displayed.
Strict
Compatibility
The syntax tree is different, users' programs may have to be modified, but not necessarily.
Quotation kit "q_ast.cmo"
All antiquotations are available.

In strict mode, better compatibility is assumed in programs using syntax tree quotations instead of syntax tree nodes. A solution is therefore to use strict mode and change the usage of nodes into the usage of quotations (which is backward compatible). But there may exist other incompatibilities.

Which mode is installed ?

To know in which mode an installed version of Camlp5 is installed, type:

  camlp5 -pmode

Selecting mode when compiling Camlp5

When compiling Camlp5 from its sources, the mode must be selected at configuration time, before doing anything else. The configure must be run with one of these options:

  ./configure -strict
  ./configure -transitional

The default is "transitional", i.e. without option, the sources are compiled in transitional mode.
